# Members Management - Target State

> **Route**: `/:slug/projects/:key/settings` (embedded) or `/:slug/members` (standalone)
> **Reference**: Linear team settings, Notion workspace members
> **Goal**: Clean member list, efficient add flow

---

## Context

Members management can exist in two contexts:
1. **Project-scoped** (current): Embedded in project settings
2. **Org-scoped** (future): Standalone page for organization members

This spec covers both, with focus on the component that can be reused.

## Behavior

### Add Member
1. Click "Add Member"
2. Form appears (inline or modal)
3. Enter email, select role
4. Submit
5. If user exists: added immediately
6. If user doesn't exist: sends invite email
7. Toast success, form closes

### Change Role
1. Click role dropdown
2. Select new role
3. Saves immediately (optimistic)
4. Toast on error only

### Remove Member
1. Click "Remove"
2. Confirm dialog appears
3. Confirm removes member
4. Toast success

### Protections
- Can't change/remove owner
- Can't remove yourself (? or can?)
- Need admin role to manage members
